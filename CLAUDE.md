## Project Overview

**Sts2SeedRoller** (杀戮尖塔自定义roll种器) is a standalone tool that reproduces Slay the Spire 2's Neow opening rewards and Act 2/3 Ancient selection RNG logic without depending on Godot or the official MegaCrit assemblies. All game logic is reimplemented in pure C# using data extracted from the game's source dump.

## Build Commands

```powershell
# Build entire solution
dotnet build Sts2SeedRoller.sln

# Run CLI
dotnet run --project src/SeedCli/SeedCli.csproj -- <args>

# Run UI
dotnet run --project src/SeedUi/SeedUi.csproj

# Run DataExtractor
dotnet run --project src/DataExtractor/DataExtractor.csproj -- --source "Slay the Spire 2 源码/seed_info.json" --output data/neow/options.json

# Rebuild Act 2/3 data from game sources
python build_sts2_act_data.py
```

The `Directory.Build.targets` automatically copies `data/` into the output directory after each build, so all projects have runtime access to the JSON datasets without manual steps.

## Architecture

### Data Flow

```
Game Source Dump (seed_info.json, C# models)
    → DataExtractor (builds data/neow/options.json)
    → Python scripts (build data/sts2/acts.json, data/ancients/options.json)
    → SeedModel (runtime library)
    → SeedCli + SeedUi (consumers)
```

### SeedModel Library Structure

The core logic lives in `src/SeedModel/` with these key areas:

| Namespace | Purpose |
|---|---|
| `Rng/` | `GameRng` — deterministic RNG replicating the game's random behavior. Counter-based, supports `FastForward()` to jump to a specific counter state. Methods: `NextBool`, `NextInt`, `NextDouble`, `NextFloat`, `Shuffle`. |
| `Collections/` | `GrabBag<T>` — weighted random selector with conditional filtering. |
| `Neow/` | `NeowGenerator` — Act 1 Neow option generation: builds negative pool, applies constraints, builds positive pool, selects 2 options, previews rewards. |
| `Events/` | Event registry and types for pluggable event generators (`SeedEventRegistry`, `SeedEventType`, `ISeedEventGenerator`). |
| `Run/` | `SeedRunEvaluator`, `SeedRunFilter` — combined evaluator for full-seed matching with Act 2/3 support. |
| `Sts2/` | `Sts2RunSimulator` — replicates `RunManager.GenerateRooms` and `ActModel.GenerateRooms` RNG consumption for Act 2/3 room generation, events, encounters, bosses, and Ancients. |

### Seed Formatting

Seeds are 10-character alphanumeric strings (alphabet: `0123456789ABCDEFGHJKLMNPQRSTUVWXYZ`, excluding `O` and `I`). Convert via `SeedFormatter.ToUIntSeed()` to `uint`. Both numeric (decimal/hex) and text formats are accepted.

### Data Files

| File | Purpose |
|---|---|
| `data/neow/options.json` | Neow options, cards, potions, pools, and metadata for Act 1 |
| `data/sts2/acts.json` | Act structures, events, encounters, bosses, ancient lists (generated by `build_sts2_act_data.py`) |
| `data/ancients/options.json` | Ancient relic options with titles and descriptions |

## Key Patterns

- **Event Generators**: Implement `ISeedEventGenerator<Context, Result>` and register via `SeedEventRegistry`. The evaluator iterates registered generators to produce full run simulations.
- **Constraint Rules**: Neow options interact via constraint rules (e.g., `CURSED_PEARL` removes `GOLDEN_PEARL` from the pool). These are encoded in `NeowOptionFilter` and applied before selection.
- **Multi-player Support**: `--players N` adjusts option pools (adds `MASSIVE_SCROLL` for >1 player, `SILVER_CRUcible` for single player, enables Scroll Boxes conditionally).

## UI Architecture

SeedUi uses WPF with HandyControl. The main ViewModel (`MainWindowViewModel`) follows MVVM with `AsyncRelayCommand` for long-running Roll operations. Configuration persists to `config.json` in the app directory.
